## 專案背景

使用者參與「星光黃金八套餐」健康挑戰群組，每日需完成 8 項任務並打卡記錄。
系統供 21–50 位成員每日回報、追蹤個人達成率，並提供管理員月結與罰款計算功能。

## 遊戲規則

### 八項每日任務（各 1 分，最高 8 分/天）

| # | 任務 | icon | 說明 |
|---|------|------|------|
| 1 | 早睡早起 | 🌙 | 12 前入睡、早上 7 前起床 |
| 2 | 破曉打拳 | 🥊 | 日出後 12 分鐘為建議開始時間（有連續天數統計） |
| 3 | 丹氣跑步 | 🏃 | 15 分鐘，雨天可室內 |
| 4 | 曬太陽 | ☀️ | 有照到才算，陰雨天不計分 |
| 5 | 工作 8 小時 | 💼 | 月累計 160 小時以上 |
| 6 | 不吃肉 | 🥗 | 各素食級別皆可 |
| 7 | 寫觀心書 | 📖 | 一階可寫覺察日記 |
| 8 | 做淨心功法 | 🧘 | 睡前效果更好 |

### 連續破曉打拳加分

- 昨天有打拳 + 今天也打拳 → 今天額外 **+0.5 分**（連續 2 天即觸發）
- 中斷一天後重新計算連續天數
- **每月「破曉王」統計**：當月連續打拳最長天數排名

> ⚠️ **目前 +0.5 加分功能已暫停實作**：`punch_bonus` 欄位固定寫入 0，`total_score = base_score`。連續天數（`punch_streak`）仍正常記錄，供破曉王統計使用。

### 達成率計算

```
起算計分日  = effective_start_date（新規則）或 join_date（既有會員 fallback）
有效起始日  = max(月份第一天, 起算計分日)
有效滿分    = 有效天數 × 8
達成率      = 累計總分 ÷ 有效滿分 × 100%
```

**新成員「起算計分日」規則（`computeEffectiveStartDate`）：**
- 台北時間 **< 12:00 加入** → 起算日 = 加入日 **+1**
- 台北時間 **≥ 12:00 加入** → 起算日 = 加入日 **+2**

> 理由：加入當天本來就難以完成 8 項任務；且中午 12:00 是打卡邏輯日邊界，若起算日等於加入日會讓中午前首次打卡記到前一日、分母卻不含該日，產生達成率異常。`/api/checkin/submit` 現已驗證「打卡邏輯日 < 起算日 → 409 拒絕」。

### 戰士階梯與罰款

| 階梯 | 達成率目標 | 未達罰款 |
|------|-----------|---------|
| 黃金戰士 | 80% 以上 | NT$200 |
| 白銀戰士 | 70% 以上 | NT$300 |
| 青銅戰士 | 60% 以上 | NT$400 |

### 打卡日邊界規則

- 打卡邏輯日以台北時間 **12:00** 為邊界：12:00 前仍屬前一日打卡窗口
  - 例：4/21 10:00 開啟頁面 → 打卡記為 **4/20**；4/21 12:01 → 記為 **4/21**
- 超過中午即自動翻頁，無獨立「補報」概念
- 已有記錄者拒絕重複提交（回 409）

### 每月階梯切換

- 25 日後才開放「選擇下月階梯」入口
- 月結時自動套用 `next_level`，並清空欄位

### 認證流程

```
登入識別：姓名 + 完整手機號碼（09 開頭，10 位）+ 密碼（若已設定）
密碼：scrypt hash（lib/password.ts）；首次登入可留空，登入後強制導向 /setup-password
失敗鎖定：連續失敗 5 次 → 鎖定 15 分鐘（failed_attempts / locked_until）
  - 多筆同名但手機錯誤時不累加 DB 計數（由 per-IP rate limit 擋 DoS）
舊帳號相容：首次以完整號碼登入時，系統自動遷移 phone_full
成功後：簽發 JWT HS256（7d TTL，含 sub/isAdmin/tv）→ httpOnly Cookie
Sliding renewal：/api/auth/me 每次呼叫會重新簽發 7 天 cookie
每支受保護 API 呼叫 getCurrentMember() → 驗證 token + 比對 token_version → 查 members 表
Admin API 呼叫 requireAdmin() → 每次查 DB 確認 is_admin（不依賴 JWT payload）
前端保護：(main)/layout.tsx 呼叫 DB 驗證，未認證跳轉 /；未設密碼跳轉 /setup-password
LINE 登入：/api/auth/line/login 產生授權 URL → LINE 回呼 /callback → 依 line_user_id 找會員
```

## 頁面功能現況

### / — 登入 / 註冊

- 兩 Tab（登入 / 加入）
- 登入：姓名 + 完整手機號碼（10 位）+ 密碼 → JWT Cookie → redirect /checkin
  - 首次登入可留空密碼；登入後自動導向 /setup-password 強制設定
- 加入：姓名、完整手機號碼、加入日期、選擇階梯 → 建立帳號（密碼於後續設定）
- 「LINE 快速登入」按鈕（所有人可見；已綁定者直接登入，未綁定回到登入頁附錯誤碼）

### /setup-password — 強制設密碼

- `(main)/layout.tsx` 偵測 `password_hash` 為 NULL 時自動導向
