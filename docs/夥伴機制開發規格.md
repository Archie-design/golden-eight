# 夥伴機制（黃金定課夥伴）

## Context

為增加定課意願與相互扶持的精神，新增夥伴機制。使用者可以互相邀請組成「黃金定課夥伴」，在主頁面看到夥伴的打卡狀況，並有鼓勵互動與相應成就系統，設計參考多鄰國好友聯盟功能。

使用者確認規格：
- **邀請**：雙向確認（被邀請方必須接受）
- **顯示**：今日是否打卡 + 本月達成率/得分 + 目前連續天數 + 完成的任務項目
- **互動**：鼓勵功能（每人每夥伴每天一次）
- **成就**：社交類 + 競爭類 + 同步類

## 資料庫 Schema

### 新增 `partner_requests` 表

```sql
CREATE TABLE partner_requests (
  id           BIGSERIAL PRIMARY KEY,
  requester_id TEXT NOT NULL REFERENCES members(id),
  target_id    TEXT NOT NULL REFERENCES members(id),
  status       TEXT NOT NULL DEFAULT 'pending',  -- 'pending' | 'accepted' | 'rejected'
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(requester_id, target_id),
  CHECK(requester_id <> target_id)
);
CREATE INDEX idx_partner_requests_target ON partner_requests(target_id);
CREATE INDEX idx_partner_requests_accepted
  ON partner_requests(requester_id, target_id) WHERE status = 'accepted';
```

### 新增 `encouragements` 表

```sql
CREATE TABLE encouragements (
  id         BIGSERIAL PRIMARY KEY,
  from_id    TEXT NOT NULL REFERENCES members(id),
  to_id      TEXT NOT NULL REFERENCES members(id),
  date       DATE NOT NULL,            -- 台北打卡日（getTodayTaipei()）
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(from_id, to_id, date)
);
CREATE INDEX idx_encouragements_to_date ON encouragements(to_id, date);
```

## API Routes

| 路由 | 方法 | 說明 |
|------|------|------|
| `/api/partners` | GET | 我的夥伴清單＋今日快照 |
| `/api/partners/search?q=` | GET | 依姓名搜尋可邀請成員 |
| `/api/partners/invite` | POST | 送出邀請 `{ targetId }` |
| `/api/partners/invitations` | GET | 待處理邀請（我送出 + 我收到）|
| `/api/partners/invitations/[id]` | PATCH | 接受/拒絕 `{ action: 'accept'\|'reject' }` |
| `/api/partners/[id]` | DELETE | 解除夥伴關係（`id` = partner_requests.id） |
| `/api/partners/[id]/encourage` | POST | 送出鼓勵（每夥伴每日一次）|

### GET /api/partners — 回傳結構

```ts
type PartnerCard = {
  requestId: number        // partner_requests.id，用於 DELETE
  memberId: string
  name: string
  level: string
  pictureUrl: string | null
  todayChecked: boolean
  todayTasks: boolean[] | null  // null = 尚未打卡
  monthRate: number             // 0-100
  monthScore: number
  punchStreak: number
  encouragedToday: boolean      // 我今天已鼓勵過他
  cheeredByToday: boolean       // 他今天鼓勵過我
}
```

查詢邏輯：
1. 從 `partner_requests` 取出我的所有 `status='accepted'` 關係（我是 requester 或 target）
2. 批次取夥伴的 `members`、`checkin_records`（今日＋本月）、`encouragements`（今日）
3. `punchStreak` 取夥伴最新打卡紀錄的 `punch_streak`

### 業務規則（API 層強制）

- 最多 10 位夥伴（接受時檢查雙方人數）
- 最多 20 個待發出邀請
- 不可邀請自己（DB CHECK 約束）
- 不可重複邀請（UNIQUE 約束）
- 若對方已先邀請我：邀請 API 回傳 409 + 提示「對方已邀請你，請至邀請管理頁接受」
- 鼓勵：UNIQUE 約束自動防止重複

## 驗證

1. **邀請流程**：A 搜尋 B 並送出邀請 → B 在邀請管理頁看到並接受 → 雙方夥伴清單出現對方
2. **重複邀請保護**：A 送出邀請後再送一次 → 409 錯誤；B 先邀請 A，A 嘗試邀請 B → 409 + 提示接受
3. **夥伴資訊顯示**：B 打卡後 A 重新整理 → 夥伴卡片顯示今日已打卡 + 今日任務
4. **鼓勵**：A 點鼓勵 B → 按鈕 disabled；同日再點 → 已防止（UNIQUE）
5. **解除關係**：A 解除 B → 雙方清單移除（DELETE partner_requests 列）
6. **成就觸發**：接受第一位夥伴 → 解鎖「初結夥伴」；A 打卡且 punch_streak > B → 解鎖「連續超越」
7. `npx tsc --noEmit` 通過
