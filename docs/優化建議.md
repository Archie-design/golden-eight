## 1. 任務圖示壓縮（最高 ROI）

### 現況

```
public/icons/tasks/  PNG 2048×2048
  task-1.png  2.0 MB     task-5.png  2.2 MB
  task-2.png  2.3 MB     task-6.png  1.5 MB
  task-3.png  1.5 MB     task-7.png  3.2 MB
  task-4.png  4.5 MB     task-8.png  3.3 MB
  ────────────────────────────────────
  total      ≈ 20 MB
```

### 問題

- 打卡頁一次需下載 8 張，**首屏 20 MB**
- 圖示透過 `mix-blend-mode: multiply` 將白底融入背景，**不需要透明通道** → 不必用 PNG
- 顯示尺寸：列表 40×40px、popup 208×208px → 2048×2048 為 **5–10 倍冗餘**

### 修復

1. 用 macOS 內建 `sips` 將 PNG 縮為 512×512、轉為 JPEG q85
2. 保留 `.png` 副檔名雖可，但統一改為 `.jpg` 較直觀
3. 更新 [lib/constants.ts](lib/constants.ts) 與 [lib/icons.tsx](lib/icons.tsx) 路徑

預期：每張 50–200 KB，**總計 < 1.5 MB**（節省 90%+）。

## 3. `members.status` 部分索引

### 現況

全站 8 處 `WHERE status = '活躍'`，但 schema 沒有對應索引。

### 修復

新增 `supabase/migrations/009_members_status_index.sql`：

```sql
CREATE INDEX IF NOT EXISTS idx_members_status_active
  ON members(status) WHERE status = '活躍';
```

部分索引（partial index）只索引活躍成員，**體積最小、查詢最快**。

## 5. Dashboard fetch 加 AbortController

### 現況

[app/(main)/dashboard/page.tsx:115-122](app/(main)/dashboard/page.tsx#L115-L122) 兩支 fetch 沒有取消機制。

### 修復

```tsx
useEffect(() => {
  const ac = new AbortController()
  Promise.all([
    fetch('/api/stats/dashboard', { signal: ac.signal }).then(r => r.json()),
    fetch('/api/stats/history',   { signal: ac.signal }).then(r => r.json()),
  ]).then(([dash, hist]) => {
    if (dash.ok) setData(dash); else toast.error(dash.msg)
    if (hist.ok) setHistory(hist.history)
  }).catch(e => { if (e.name !== 'AbortError') console.error(e) })
  return () => ac.abort()
}, [])
```

## 10. SW 快取任務圖示

### 修復

[public/sw.js](public/sw.js) 加入 cache-first 策略：

```js
self.addEventListener('fetch', e => {
  if (e.request.url.includes('/icons/tasks/')) {
    e.respondWith(
      caches.match(e.request).then(r =>
        r || fetch(e.request).then(resp => {
          const clone = resp.clone()
          caches.open('icons-v1').then(c => c.put(e.request, clone))
          return resp
        })
      )
    )
  }
})
```
