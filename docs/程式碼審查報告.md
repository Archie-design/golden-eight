## 部署操作（套用前必做）

1. **套用 DB migration**：於 Supabase SQL Editor 執行
   `supabase/migrations/003_security_hardening.sql`
2. **環境變數新增**（建議）：
   - `PHONE_PEPPER`：長度 ≥ 32 字元，用於 HMAC(phone)。未設定時自動沿用 `JWT_SECRET`（功能可用，但輪替 JWT 會同時失效手機比對；生產環境應明確設定）。
3. **既有帳號的 `phone_hash` 會於下次登入時漸進遷移**，無需手動處理。

## 詳細修復內容

### P0-1 登入同名連鎖鎖定
**檔案**：`app/api/auth/login/route.ts`
**修復**：
- 失敗不再對「所有同名成員」累加 `failed_attempts`；僅當同名只有一筆時才對該筆累加。
- 新增 per-IP rate limit（10 次/分鐘），作為主要防線。
- 鎖定回應補上 `Retry-After` header。

### P0-2 排程模板 delete-then-insert 無交易
**檔案**：`app/api/schedule/template/route.ts`, `supabase/migrations/003_security_hardening.sql`
**修復**：新增 `replace_schedule_template(p_member_id, p_is_public, p_blocks)` Postgres function，delete + insert 於同一 transaction，任何中斷皆整筆回滾。

### P0-3 `line_user_id` 無 UNIQUE
**檔案**：`supabase/migrations/003_security_hardening.sql`
**修復**：新增 partial unique index `idx_members_line_user`，阻止任何來源寫入重複 `line_user_id`。

### P1-4 標籤 ID race
**檔案**：`app/api/schedule/tag/route.ts`
**修復**：改以 `randomUUID().slice(0, 8)` 作為 id 後綴，避免 `count+1` 讀寫間隙碰撞。

### P1-5 CSV 公式注入
**檔案**：`lib/csv.ts`（新）、`app/api/admin/export/route.ts`
**修復**：
- 新增 `csvField()` / `csvRow()`：以 `=/+/-/@/\t/\r` 起首的值前綴 `'`；含 `"`/`,`/換行者以雙引號包裹並跳脫。
- export 路由改用 `csvRow(...)`。

### P1-6 Rate Limiting
**檔案**：`lib/rate-limit.ts`（新）
**套用**：
- `/api/auth/login` — 10 次/分鐘/IP
- `/api/auth/register` — 5 次/10 分鐘/IP
- `/api/checkin/submit` — 20 次/分鐘/IP
- `/api/schedule/tag` POST — 30 次/分鐘/IP

實作備註：per-instance 記憶體計數，跨 instance 各自獨立。未來需要嚴格一致時可無痛替換為 Vercel KV / Upstash Ratelimit。

### P1-7 Sunrise DB 快取
**檔案**：`lib/sunrise.ts`, `supabase/migrations/003_security_hardening.sql`
**修復**：
- 新增 `sunrise_cache(date PK, sunrise, fetched_at)` 表。
- 取值順序：DB 命中 → 外部 API + upsert 快取 → 失敗 fallback `06:00`。
- 跨所有 lambda instance 僅第一個請求打外部 API。

### P1-8 手機號雜湊 + RLS
**檔案**：`lib/phone.ts`（新）、`app/api/auth/login/route.ts`、`app/api/auth/register/route.ts`、`app/api/admin/members/route.ts`、`supabase/migrations/003_security_hardening.sql`
**修復**：
- 新增 `phone_hash TEXT`（HMAC-SHA256，pepper 由 `PHONE_PEPPER` 或 `JWT_SECRET` 提供）。
- 登入以 hash 比對；舊資料（`phone_hash IS NULL`）以 `phone_last3` 相容路徑驗證後漸進寫入 hash。
- 新增 partial unique index `idx_members_phone_hash`。
- 啟用 `members / checkin_records / monthly_summary / achievements / tag_library / schedule_template / sunrise_cache` 的 RLS（無 policy = 預設全部拒絕；service_role 自動繞過）。

### P1-9 Cron 註解/方法
**檔案**：`app/api/cron/daily-reminder/route.ts`
**修復**：註解改為「GET」以符合實作；加註修改排程設定時應同步檢查。

### P2-10 Backfill O(n) 重構
**檔案**：`app/api/admin/backfill-achievements/route.ts`
**修復**：以 running counter（`totalCount`、`perfectCount`）與 sliding window（最後 105 日）取代「每步 slice(0, i+1)」，從 O(n²) 降至 O(n)。

### P2-11 月結批次 level update
**檔案**：`app/api/admin/settlement/route.ts`
**修復**：依目標 level 分組、以 `.in('id', ids)` 一次 update；最多 3 次 round-trip 取代 N 次。

### P2-12 Dedup sunrise
**檔案**：`app/api/checkin/today/route.ts`, `lib/sunrise.ts`
**修復**：新增 `addMinutes(hhmm, n)` 輔助；今日頁僅呼叫一次 `getSunriseTime()`，再以 `addMinutes(sunrise, 12)` 導出建議開始時間。

### P2-13 Cookie `secure` flag
**檔案**：`lib/cookie-options.ts`（新），以及：
- `app/api/auth/login/route.ts`
- `app/api/auth/register/route.ts`
- `app/api/auth/logout/route.ts`
- `app/api/auth/line/route.ts`
- `app/api/auth/line/login/route.ts`
- `app/api/auth/line/callback/route.ts`

**修復**：統一 cookie 選項；`secure: NODE_ENV === 'production'`。

### P2-14 JWT token_version
**檔案**：`lib/auth.ts`, `lib/api-helper.ts`, `types/index.ts`, `supabase/migrations/003_security_hardening.sql`, 所有 issue JWT 處
**修復**：
- `members` 新增 `token_version INT DEFAULT 0`。
- JWT payload 新增 `tv`；`getCurrentMember` / `requireAdmin` 驗證 DB 側與 token 的 `tv` 一致；不一致回 401。
- 遞增 `token_version` 即撤銷該成員所有既有 JWT（可供「強制登出」/外洩應變使用）。

### P2-15 增量成就計算
**檔案**：`lib/scoring.ts`, `app/api/checkin/submit/route.ts`
**修復**：
- 新增 `calcNewAchievementsFromAggregates({ totalCount, perfectCount, recentSorted, todayRecord, alreadyUnlocked })`。
- 打卡提交路由改用：`count('*')` + `count('*') WHERE base_score=8` + 最近 105 日紀錄，單次請求不再 SELECT 全量紀錄。
- 舊 `calcNewAchievements` 保留為相容 wrapper（被 backfill 使用）。

### P2-16 SQL-side 標籤移除
**檔案**：`app/api/schedule/tag/route.ts`, `supabase/migrations/003_security_hardening.sql`
**修復**：新增 `remove_tag_from_templates(p_member_id, p_tag_id)` RPC；以 `jsonb_array_elements` + `jsonb_agg` 一次 SQL 清除所有 schedule_template 中該標籤，取代 app 層逐列 update。

### P3-17 `getNowHourTaipei` locale
**檔案**：`lib/api-helper.ts`
**修復**：改用 `en-US` + `hour: '2-digit'` + `hour12: false`，並把 `"24"` 正規化為 0，消除不同 ICU 版本差異。

### P3-18 `Retry-After`
**檔案**：`lib/rate-limit.ts`、`app/api/auth/login/route.ts`
**修復**：所有 429 回應皆附 `Retry-After` header（秒數）。

### P3-19 schema.sql 與 migrations 分裂
**檔案**：`supabase/schema.sql`
**修復**：重寫為「當前真相」快照，與 migrations 002/003 對齊（phone_last3 改為選填、block_tags JSONB、新增 phone_hash/token_version/sunrise_cache、unique indexes、RLS、RPC）。新環境直接執行此檔即可 bootstrap。

### P3-20 成就寫入失敗靜默消除
**檔案**：`app/api/checkin/submit/route.ts`
**修復**：insert 失敗改 `console.error` 並回傳空 `newAchievements` 陣列；使用者端仍正常顯示打卡成功，管理員可由 log 察覺。

## 後續建議（非本次審查範疇）

1. **rate limiter 升級為 KV**：單一 instance 的記憶體計數在 Vercel Fluid Compute 夠用，但如需多 region 嚴格一致，可導入 `@upstash/ratelimit` + Vercel KV。
2. **JWT 縮短 + 滑動續期**：目前 30 天仍長。短期可將 `maxAge` 改為 7 天 + `/api/auth/me` 續期。
3. **觀測性**：若需生產追蹤，可接 Sentry 或 Vercel Observability；本次未動此部分。
4. **React 19 lint 警告**：`set-state-in-effect` 警告應逐一修正（屬既存問題；`app/page.tsx`、`app/(main)/schedule/page.tsx`、`components/schedule/TimelineView.tsx`）。
